  • TSDKCore

    Main telerion SDK class. Must be used as telerion SDK entry point and linked to telerion service via init(‘telerion-URL’) before any other actions. Any outgoing or incoming session requires to be authenticated so make authentication first via login(…). SDK keeps authentication while app is alive or till logout( ) is called.

  • TSDKSession

    The TSDKSession interface represents a signaling and media session between client and telerion service. Session can be call or chat. Both chat and call sessions includes caller (originator) and callee (terminator). For outgoing session login is used as caller’s number and callee’s number must be provided as parameter of connect(String) method.
